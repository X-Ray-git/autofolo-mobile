🎯

AI 智能过滤

基于 Auto Folo 的精细化 Prompt 引擎，DeepSeek 逐篇判定文章是否值得阅读。纯视觉、具身智能、币圈杂讯自动屏蔽，白名单频道和偏好作者始终保留。

🌐

自动翻译

按订阅源粒度配置自动翻译。支持 DeepSeek v4-flash / v4-pro，思考模式可选，temperature、并发数均可调。翻译结果与原文无缝切换。

📝

AI 摘要生成

所有未读文章自动生成一句话摘要，快速判断是否值得点开。v4-pro + 思考模式提供高质量提炼。

🖼️

智能图片代理

集成 Folo 官方 img.folo.is 代理，自动为微博、Pixiv、Instagram 等需要特定 Referer 的 CDN 补全请求头。画廊支持双击放大、捏合缩放、滑动关闭。

🎬

视频内联播放

Twitter 视频自动提取 poster 缩略图，点击内联播放。进度条可拖拽定位，支持暂停/播放切换，流畅不跳转外部浏览器。

📊

三级订阅源管理

View → 分类 → 订阅源 树形结构，点击箭头展开折叠，点击文字查看该层级全部文章。实时未读计数徽标，搜索即时过滤。

数据流水线

从 API 到你的屏幕

六步处理链路，每一步都可以独立配置和控制并发度。全部 LLM 调用互不干扰，16 路并行最大吞吐。

1

拉取

从 Folo API 获取 feeds / social / inbox 三类未读文章

2

过滤

DeepSeek JSON Output 逐篇判定保留/拒绝，被拒进入审核队列

3

翻译

按源自动翻译开关，仅翻译保留文章，节省 API 费用

4

摘要

全部未读文章生成一句话摘要，快速浏览核心内容

5

审核

左右滑手势审核被拒文章，捞回或确认拒绝并自动标已读

6

同步

已读状态云端同步，失败自动重试 5 次，始终与服务器一致

Hive 数据模型

6 个 Box，上限 5,000 篇文章，同步写入。

ArticleModel (18 fields)
  entryId / feedId / feedTitle / feedImage / title / url / content
  publishedAt / isRead / category / subscriptionCategory / author
  imageUrl / isRejectedByAi / filterReason / filterReviewed

Box: articleDb    — 文章持久化 (limit 5,000)
Box: readStatus   — 已读状态标记 (双向同步源)
Box: setting      — LLM配置 / Token / Filter Prompt
Box: localCache   — 订阅源缓存 (TTL 30min, key: v2)
Box: translations — 翻译记录
Box: summaries    — 摘要记录
